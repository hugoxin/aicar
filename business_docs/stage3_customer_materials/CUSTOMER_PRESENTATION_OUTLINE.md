# 客户演示PPT大纲

## 页1：项目标题

- 页面标题：AI智能无人洗车仿真系统
- 核心观点：这是一个从车辆识别到洗车仿真和可视化展示的软件原型。
- 建议放什么图：Stage3.3 客户展示页首屏截图。
- 讲解要点：先说明当前是软件闭环，不是硬件控制。

## 页2：行业痛点

- 页面标题：无人洗车研发为什么容易试错
- 核心观点：车辆差异、喷嘴布局、流程控制和硬件联调都容易带来成本。
- 建议放什么图：传统研发流程和软件先行流程对比图。
- 讲解要点：直接上硬件会让问题发现得太晚。

## 页3：我们的思路：先软件仿真，再硬件联调

- 页面标题：先把逻辑跑通，再接设备
- 核心观点：先用软件验证识别、策略、路径和覆盖，再进入机械/PLC。
- 建议放什么图：Stage1 -> Stage2 -> Stage3 -> Stage4/5路线图。
- 讲解要点：降低试错成本，统一技术口径。

## 页4：系统总体架构

- 页面标题：从图片到洗车仿真的完整链路
- 核心观点：车辆识别结果会驱动后续策略、空间、喷嘴、流程和可视化。
- 建议放什么图：车辆图片到 `vehicle_type_result.json` 再到 Stage2/Stage3 的流程图。
- 讲解要点：强调模块边界清晰，接口用 JSON 串联。

## 页5：阶段1车辆识别

- 页面标题：AI车辆识别小闭环
- 核心观点：系统可以从图片输出 `sedan` / `suv` / `mpv`。
- 建议放什么图：车辆检测框、裁切图、识别结果摘要。
- 讲解要点：当前是小样本 demo，不代表商业级识别精度。

## 页6：阶段2洗车仿真主链路

- 页面标题：洗车策略、空间、喷嘴、流程和路径
- 核心观点：阶段2把车型结果转成可检查的仿真 JSON。
- 建议放什么图：Stage2 Pipeline Demo 页面或 JSON 输出关系图。
- 讲解要点：强调当前是抽象仿真，不是真实机械控制。

## 页7：阶段3可视化展示

- 页面标题：让仿真结果可看、可讲、可复盘
- 核心观点：2D可视化和时间轴动画让流程和覆盖更直观。
- 建议放什么图：2D报告和时间轴动画截图。
- 讲解要点：展示层不改变核心算法，只帮助沟通。

## 页8：当前 Demo 指标

- 页面标题：当前样例运行结果
- 核心观点：当前样例车型为 `sedan`，覆盖估算通过。
- 建议放什么图：指标卡片。
- 讲解要点：说明 `coverage_pass=true` 是仿真估算通过，不是实车洗净率。

## 页9：客户价值

- 页面标题：为什么这件事值得做
- 核心观点：降低试错、提前验证、便于汇报和后续硬件对接。
- 建议放什么图：价值卡片或三类客户收益。
- 讲解要点：分别讲设备厂商、运营方、研发团队的价值。

## 页10：技术边界

- 页面标题：当前能说什么，不能说什么
- 核心观点：对外表达要准确，避免把软件仿真说成硬件控制。
- 建议放什么图：可以说/不能说两列表格。
- 讲解要点：边界清楚反而更可信。

## 页11：后续路线

- 页面标题：从软件闭环走向真实系统
- 核心观点：Stage4 真实路径规划，Stage5 PLC/硬件，Stage6 商业后台。
- 建议放什么图：阶段路线图。
- 讲解要点：说明每一步的目标和投入重点。

## 页12：合作方向

- 页面标题：可继续合作的方向
- 核心观点：可以围绕路径规划、机械联调、设备数据和商业化系统继续推进。
- 建议放什么图：合作模块清单。
- 讲解要点：把下一步需求拆成可评估、可报价、可交付的模块。
