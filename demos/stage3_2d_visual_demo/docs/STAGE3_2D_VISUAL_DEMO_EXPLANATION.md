# Stage3 2D Visual Demo Explanation

## 1. 阶段3.1增加了什么

阶段3.1在阶段2仿真链路基础上增加了单文件 HTML 2D 可视化报告。它读取阶段2生成的策略、空间、喷嘴、流程、抽象路径和覆盖率 JSON，把这些数据画成俯视图、侧视图、表格和时间线。

## 2. 2D图能展示什么

- 洗车房边界。
- 车辆 `bounding_box`。
- 安全包络 `safe_envelope`。
- 车顶、左右侧、前后、轮毂区域。
- 抽象喷嘴路径点和 path segment。
- 区域覆盖率、喷嘴分配和流程时间线。

## 3. 为什么先做2D

2D 可视化可以更快验证阶段2数据是否连贯：车辆是否在洗车房内、路径点是否落在车辆周围、覆盖率表是否能对应到区域和流程。它比 3D 和动画更轻，更适合冻结前检查数据链路。

## 4. 当前能证明什么

当前能证明阶段1识别结果已经可以驱动阶段2仿真 JSON，并进一步形成可读的 2D 展示报告。报告能帮助项目组快速检查空间、喷嘴、流程、路径和覆盖率之间是否一致。

## 5. 当前不能证明什么

当前不能证明真实清洗效果、真实机械轨迹、真实喷雾覆盖、PLC 时序或硬件可靠性。所有坐标仍是仿真参考坐标。

## 6. 后续方向

- 阶段4：真实路径规划。
- 阶段5：PLC/硬件联调。
