# Stage3 Timeline Animation Demo Explanation

## 1. 阶段3.2增加了什么

阶段3.2在阶段3.1静态 2D 可视化基础上增加了时间轴播放、状态切换、当前清洗区域高亮和当前抽象路径高亮。

## 2. 时间轴动画能展示什么

- 洗车流程从 `idle` 到 `completed` 的顺序。
- 当前 `state_id`、状态名称、起止时间和持续时间。
- 当前状态对应的清洗区域。
- 当前状态对应的抽象喷嘴路径段。
- 当前状态分配的喷嘴。
- 覆盖率和路径统计。

## 3. 为什么先做轻量 HTML 动画

轻量 HTML 动画只依赖阶段2 JSON 和原生 JavaScript，便于快速验证流程、路径和覆盖率之间的关系。它比 3D 或复杂动画引擎更容易审查，也更适合作为阶段3可视化基线。

## 4. 当前能证明什么

当前能证明阶段2的 wash flow、abstract path 和 coverage report 可以被串成可播放的可视化链路，便于项目组观察每个状态对应的区域和路径。

## 5. 当前不能证明什么

当前不能证明真实机械运动、喷雾覆盖、流体效果、PLC 时序或硬件可靠性。

## 6. 后续方向

- 阶段3.F：冻结可视化基线。
- 阶段4：真实路径规划。
- 阶段5：PLC/硬件联调。
