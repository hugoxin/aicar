# 阶段2完整链路可视化 Demo 说明

本 Demo 用于项目组或客户展示阶段2从识别结果到覆盖率报告的完整软件链路。

## 阶段2已经完成什么

阶段2目前已经串联：

1. 洗车策略计划。
2. 车辆包络与洗车空间模型。
3. 喷嘴模型与覆盖参数。
4. 洗车流程状态机。
5. 抽象喷嘴路径点。
6. 抽象覆盖率检查与报告。

这些模块都输出可检查的 JSON，方便后续可视化、路径规划和工程讨论。

## 为什么阶段2不是硬件控制

阶段2仍然是软件仿真和规划前置层。它不会连接 PLC，不会控制泵、阀、伺服轴或传感器，也不会下发任何真实设备指令。

## 六个模块分别解决什么

- 阶段2.1：根据车型和 `wash_profile` 生成洗车策略。
- 阶段2.2：把车辆尺寸和洗车房空间转换为静态几何模型。
- 阶段2.3：把车辆区域映射到喷嘴和覆盖参数。
- 阶段2.4：把清洗动作组织成状态机 timeline。
- 阶段2.5：为每个状态生成抽象喷嘴路径点。
- 阶段2.6：检查每个区域是否有路径覆盖并估算覆盖率。

## 当前 Demo 能证明什么

- 阶段1识别结果可以进入阶段2。
- 阶段2.1到阶段2.6可以一键生成。
- 每个模块之间的数据字段已经能衔接。
- 覆盖率报告可以解释每个 zone 的覆盖情况。
- HTML 报告可以用于项目沟通和阶段性展示。

## 当前 Demo 不能证明什么

- 不能证明真实清洗效果。
- 不能证明真实喷嘴轨迹可执行。
- 不能证明水流、压力、碰撞、安全联锁或传感器反馈正确。
- 不能替代 PLC 或硬件联调。
