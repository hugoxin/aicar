# Viewer V1.0.1 视觉修补

## 定位

V1.0.1 只修补 Stage4 三维离线 Viewer 的视觉理解层，不修改 Stage4.5-R 正式路径、坐标、时间戳、状态语义或安全结论。

## V1 视觉评审发现

- 扫描点容易被完整轨迹淹没。
- 当前段、已执行段、未来段和辅助连接线的视觉权重过于接近。
- 右侧面板默认显示过多技术 ID，不利于客户和非技术人员理解。
- 高密度状态路径会与车辆轮廓争夺注意力。

## 修补内容

- 扫描点增大，并增加状态色光环、缓慢脉冲和始终面向相机的准星。
- 路径按当前段、已执行、未来主扫描、辅助连接和未知角色分层显示。
- 当前段使用双层线条高亮；尾迹加长并按状态着色、由暗到亮。
- “聚焦当前状态”默认开启，非当前状态自动降噪。
- 辅助连接线默认隐藏，可通过开关恢复；辅助线仅表示扫描区域之间的移动。
- 点标签只显示中文状态和区域。
- 右侧面板分为演示信息与默认折叠的技术详情，长 ID 使用紧凑等宽显示和悬停完整值。
- 页面标题、数据来源、当前模式和执行说明改为中文优先。

## 数据与边界

- 正式 source/viewer point count 仍为 2503。
- 状态 7 个、区域 6 个、路径长度 328.502099 m、候选时长约 2570.903 s。
- 主扫描与辅助连接角色只在浏览器加载时派生，不写回 `viewer_scene.json`。
- Viewer 仍使用通用 MPV 演示映射，不是真实车型 CAD 或标定路径。
- 页面仍是离线仿真展示，不连接 PLC、伺服或真实设备，也不生成控制指令。
