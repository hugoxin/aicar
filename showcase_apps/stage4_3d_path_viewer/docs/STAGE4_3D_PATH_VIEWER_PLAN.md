# Stage4 三维轨迹 Viewer 方案

## 定位

Stage4.5-R 已形成连续清洗面路径，但 JSON 与工程报告不适合直接演示。Viewer 在底层算法之外增加独立展示层，把状态、区域、路径点和时间参数转换为可交互三维画面，不改变原算法与冻结结果。

## 数据流

```text
Stage4.5-R machine/surface path
  -> export_viewer_scene.py
  -> public/data/viewer_scene.json
  -> Three.js
  -> 通用MPV + 彩色路径 + 发光扫描点
```

## 展示方式

- 车辆：由基础几何体组合的品牌无关通用 MPV，支持半透明实体、线框、实体加线框。
- 路径：按清洗状态着色，transition 单独显示，已执行路径和当前段高亮。
- 动画：按时间戳线性插值，支持播放、暂停、重置、进度拖动、倍速、状态跳转和可选循环。
- 相机：OrbitControls 与斜视、顶视、左右、前后预设，预设切换使用平滑过渡。

## Display Transform

Stage4.5-R 使用冻结参考车辆坐标。V1 按长宽高轴分别缩放到通用 MPV 展示尺寸，同时在数据中保留 source/display 坐标、缩放和位移。该变换仅用于展示扫描逻辑，不是 MPV 重新规划，不继承原安全结论。

## 当前边界

- 当前不是真实 MPV 标定轨迹。
- 当前扫描点代表未来喷头中心，不是喷头机械模型。
- 不连接 PLC、伺服、机器人或真实设备。
- Stage4.6 仍在独立分支，V1 不依赖其几何姿态输出。

## 后续可选方向

未来可独立评审 Stage4.6 数据适配、真实喷头简化模型、法向箭头、安全距离、多执行机构和碰撞区域显示。本版本不自动进入这些工作。
