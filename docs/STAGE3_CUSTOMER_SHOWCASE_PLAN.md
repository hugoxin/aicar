# Stage3 Customer Showcase Plan

## 1. 阶段3.3定位

阶段3.3是客户展示页优化阶段。它不是新算法，不是 3D，不是复杂动画引擎，不是真实路径规划，也不连接 PLC 或硬件。

## 2. 为什么需要客户展示页

阶段1、阶段2和阶段3已经形成多个工程 Demo，但这些页面偏技术说明。客户展示页把成果包装成项目组、客户、领导、合作方和申报材料都能理解的一页式页面。

## 3. 与 Stage3.1 / Stage3.2 的关系

- Stage3.1：生成 2D 静态可视化报告。
- Stage3.2：生成时间轴动画报告。
- Stage3.3：把已有能力整理成面向客户的价值说明、阶段进度、能力边界和后续路线。

Stage3.3不改变 `stage3-visual-baseline` tag，只是在冻结基线后追加展示优化。

## 4. 页面展示内容

- 项目定位。
- 当前能力。
- 完整链路。
- Demo 数据摘要。
- 项目阶段进度。
- 业务价值。
- 技术边界。
- 已有 Demo 入口。
- 下一步建议。

## 5. 当前能证明什么

当前能证明 AI 智能无人洗车软件闭环已经贯通：车辆识别结果可以驱动洗车策略、空间模型、喷嘴覆盖、流程状态机、抽象路径、覆盖率报告和可视化展示。

## 6. 当前不能证明什么

当前不能证明真实清洗效果、真实机械轨迹、真实流体仿真、PLC 控制或硬件可靠性。

## 7. 适用场景

- 客户沟通。
- 项目汇报。
- 领导评审。
- 申报材料准备。
- 合作方技术交流。

## 8. 后续方向

后续可以进入 Stage4 真实路径规划与运动约束，也可以继续做客户版 UI 和视频化展示。
