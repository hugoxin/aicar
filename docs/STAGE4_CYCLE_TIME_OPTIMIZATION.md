# 阶段4.4：周期与调度优化

## 1. 两种时间

`estimated_motion_duration_s` 是单条机械候选轨迹按速度、加速度和采样规则重新参数化后的结束时间。`total_schedule_duration_s` 是三个执行机构在state顺序和资源互锁下完成全部task的总周期，两者不是同一指标。

## 2. 累计任务延迟

`total_delay_s` 是多个任务因资源冲突产生的等待累计值。同一时间段可能同时影响多个后续task，因此累计延迟可以大于总周期。阶段4.3级联平移会把一次冲突传播到同一执行器后续队列，基线累计值为20365.367 s。

## 3. Earliest Feasible Start

阶段4.4尝试为冲突task查找最早可用时间，只延迟实际受阻task；其他执行器在资源不冲突时继续运行。资源释放后立即尝试调度，并保留配置中的安全时间margin。

## 4. Local Swap 与并行

局部交换只发生在同一wash state内，并受task依赖、执行器顺序和资源锁约束。不同执行器且扫掠空间不冲突的task可并行。任何交换不得改变task集合、state总体顺序、zone或nozzle语义。

## 5. 左右同步

`side_clean`和`air_dry`会尝试同起点或1秒内近同步。若左右任务共享中心空间、扫掠AABB边界接触或同步候选使综合周期目标变差，可以保持`BLOCKED_BY_INTERLOCK`。同步数量是优化目标，不是强制通过条件。

## 6. 资源锁

中心共享空间和front/rear crossing按不同执行机构的时间区间互斥；左右和顶部轨道分别归属对应执行器。边界接触按冲突处理，不能为了缩短周期删除互锁。优化结果必须保持`conflict_count_after_resolution=0`和`unresolved_conflict_count=0`。

## 7. 指标与边界

报告对比运动时间、调度周期、累计延迟、并行组、同步组和冲突数量。不能通过删除任务、降低250 mm安全下限、减少检查或伪造同步来改善指标。当前结果是启发式参考，不含真实控制器周期、PLC扫描时间、伺服响应或硬件延迟。
